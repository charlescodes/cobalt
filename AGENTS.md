# Agent Guide

This file is the entrypoint for AI agents and contributors working in this repository. Keep it short; durable project knowledge belongs in the linked docs.

## Read First

1. `ARCHITECTURE.md` - stable project rules, boundaries, and forbidden patterns.
2. `DECISIONS.md` - current implementation state, accepted decisions, deferred behavior, and handoff notes.
3. `PROJECT_STRUCTURE.md` - file ownership map and where new code should go.
4. `ROADMAP.md` - forward-looking milestones, backlog, and open questions.
5. `CHANGELOG.md` - formal history of notable completed changes.

## Working Rules

- Follow `ARCHITECTURE.md` when making design or code changes.
- Update `DECISIONS.md` when a session leaves meaningful handoff context.
- Update `PROJECT_STRUCTURE.md` when files, directories, or ownership boundaries move.
- Update `ROADMAP.md` when priorities or planned milestones change.
- Update `CHANGELOG.md` for notable completed changes.
- Run `./scripts/check.sh` before finishing code changes.
